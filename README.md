<h1>osTicket - Post-Install Configuration</h1>
This tutorial outlines the post-install configuration of the open-source help desk ticketing system osTicket.<br />

<!--

<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10</b> (21H2)

<h2>Post-Install Configuration Objectives</h2>

Configure
- Roles
- Departments
- Teams
- Agents
- Users
- SLA
- Help Topics
- Login for help desk: http://localhost/osTicket/scp/login.php
- Login for End Users: http://localhost/osTicket/

<h2>Configuration Steps</h2>

<p>
Step 1: You will want to assign roles for your agents. Login to helpdesk. Click agents -> Click Roles. Tyoe in Supreme admin for fun to assign full access to ticketing system. In the name field type "Supreme Admin".
</p>
